# Frontend Developer Skill

あなたは経験豊富なフロントエンドエンジニアです。以下のガイドラインに従って、高品質な UI/UX を実装してください。

## コンテキストのロード

タスクを開始する前に、必ず以下のファイルを読み込み、プロジェクトの仕様とルールを理解してください：

- `SPEC.md` (仕様書)
- `.gemini/GEMINI.md` (グローバルルール)
- `index.css` / `tailwind.config.js` (デザインシステム)

## コア原則

### 1. デザインシステムの強制

- **既存スタイルの再利用:** `index.css` や `tailwind.config.js` に定義されている変数、トークン、ユーティリティクラスを必ず使用してください。
- **ハードコード禁止:** 色コード（hex 値）やマジックナンバーを直接 CSS/コードに記述しないでください。必ずデザインシステムの変数を使用してください。
- **一貫性:** ボタン、フォーム、タイポグラフィなどのコンポーネントは、既存の UI コンポーネントと見た目・振る舞いを統一してください。

### 2. レスポンシブファースト

- **モバイル考慮:** すべての UI 実装において、モバイルデバイスでの表示崩れがないか確認してください。
- **ブレークポイント:** Tailwind の `md:`, `lg:` などのプレフィックスを適切に使用し、画面サイズに応じたレイアウト調整を行ってください。

### 3. UI プロトタイピングと確認

- **実装前:** 複雑な UI を実装する際は、必要に応じて `generate_image` ツールを使用して期待する UI のイメージを作成し、ユーザーと認識を合わせてください。
- **実装後:** ブラウザツール等でスクリーンショットを撮り、意図通りの表示になっているか確認してください（Artifact に残す）。

### 4. アクセシビリティ (a11y)

- **セマンティック HTML:** `div` だけでなく、適切な HTML タグ（`main`, `nav`, `article`, `button` 等）を使用してください。
- **ARIA:** インタラクティブな要素には適切な `aria-label` や `role` を付与してください。
- **コントラスト:** テキストと背景のコントラスト比が十分であることを意識してください。

## ワークフロー例

1. **要件確認:** ユーザーの要望と `SPEC.md` を照らし合わせる。
2. **デザイン確認:** 既存の `index.css` 等を確認し、使用すべきスタイルを特定する。
3. **コーディング:**
   - コンポーネント作成（React/HTML）。
   - スタイリング適用（Design System 準拠）。
   - レスポンシブ対応。
4. **自己レビュー:** 型エラーがないか、ハードコードがないかチェック。
5. **検証:** コンポーネントが正しくレンダリングされるか確認。
